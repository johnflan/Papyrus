Project Proposal

Title: A Scalable Distributed Object Store

Offered to:
523 - MAC Individual Project
542 - MSc Computing Science (Specialist) Individual Project - M.Sc in Computing (Distributed Systems) - pcsds5
542 - MSc Computing Science (Specialist) Individual Project - M.Sc in Computing (Software Engineering) - pcsse5
543 - MRes Individual Project

Description:

Global web applications (eg Facebook and Youtube) must store a large amount of user-generated content such as photos and videos and retrieve it quickly based on user requests. For example, Facebook must handle 60 terabytes of new image data each week. Web applications use specialised databases that act as distributed object stores for storing and retrieving this data.



The scalability and performance of object stores often becomes a bottleneck that governs the performance of the entire web application. As a result, object stores use sophisticated caching mechanism to reduce disk I/O and custom storage layout of data on disks to minimise the number of disk seeks. While a number of designs for scalable object stores have been proposed by researchers, it is an open problem to consider a store that spans multiple data centre locations efficiently.



Project Goals



The goal of this project is to design, implement and evaluate a distributed object store that can be deployed across multiple data centres. The object store should possess the following key features:




  1. Caching: it should aggressively cache user-requested content to avoid disk accesses;

  2. Custom data layout: it should lay out objects on disk in a custom fashion, for example, inspired by log-structured file systems; this will avoid multiple disk accesses when retrieving a single object;

  3. Data partitioning: it should partition objects between two (or more sites) and intelligently load-balance requests between them.


Roadmap



First, you will conduct a literature survey of existing designs of distributed objects stores. Based on this, you will identify short-comings of currently proposed architectures when they are deployed across a small number of data centres.



Second, this should lead to a design of an object store that provides good performance in a multi-site setting. A challenge in this context will be decide on a caching strategy and on a load-balancing mechanism that assign web clients to data centres. This may involve making clients more intelligent when choosing servers to send requests to (eg using JavaScript, HTML5 etc).



Finally, you should implement the complete system together with a simple, web-based workload generator and carry out an experimental evaluation. This could be done using a local cluster test-bed here at Imperial where you simulate a WAN links between machines, or using a second data centre location overseas. We could also provide resources for a cloud deployment on Amazon EC2.



Skills




  • Strong programming skills related to the implementation of distributed applications; familiarity with low-level message passing code

  • Interest in large-scale application design and load-balancing algorithms

  • Interest in experimental systems evaluation using Internet test-beds
